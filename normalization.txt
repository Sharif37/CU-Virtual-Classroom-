\section{Normalization}\label{sec:norm}
Normalization is the process of making the table free from insertion ,update and delete anamoly and save space by reducing the redundent data.\\ \\
\textbf{Types of Normalization:}
There are several types of normalization.\\
\begin{itemize}
\item First Normal Form (1F)
\item Second Normal Form (2F)
\item Third Normal Form (3F)
\item Boyce-Codd Normal Form (BCNF)\\
\end{itemize}
\textbf{First Normal Form (1F):}The table or relation is said to be in First Normal Form if it does not contain any multi-valued or composite attributes. So the table or relation should contain only single-valued attributes for fulfilling the condition for First Normal Form.\\ \\
\textbf{Second Normal Form (2F):}A relation or table to be in Second Normal Form should be in First Normal Form and it should not hold any partial dependency. So in Second Normal Form, the table should not contain any non-prime attribute depending upon the proper subset of any candidate key.\\ \\
\textbf{Third Normal Form (3F):}A table is in Third Normal Form if it is in Second Normal Form and there should not be any transitive dependency for the non-prime attributes. .\\ \\
\textbf{Boyce-Codd Normal Form (BCNF):}For a table to be in Boyce-Codd Normal Form, it should be in Third Normal Form.\\ \\ \\
\textbf{Benifits of Normalization:}\\
\begin{itemize}
\item To reduce redundency from the table
\item To save space
\item To remove insert,update and delete anamoly
\item Normalize minimize the null value
\item Normalization simplifies the query
\item It simplifies the database structure\\
\end{itemize}
